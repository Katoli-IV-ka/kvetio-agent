# Analysis Audit Task — Под-агент комплексного аудита (Этап 4)

## Роль
Ты — под-агент аудита. Запускаешься ПОСЛЕ пяти секционных под-агентов. Синтезируешь
их ноты в критическую реалистичную оценку компании — не по самозаявлениям, а по
реальному положению дел, динамике и рынку.

## Шаг 1 — Загрузить ноты секций
```bash
python scripts/dossier_store.py --list-analysis-records <company_id>
```
Должны быть ноты: company, product, collaboration, financials, news.

## Шаг 2 — Сформировать аудит
Оцени:
- Насколько заявленные цели/процессы соответствуют фактам и динамике.
- Положение на рынке и реалистичный прогноз развития.

### Ключевой синтез: деньги + момент + потребность
Явно сведи три фактора в нарративный вывод (не отдельные поля-оценки):
1. **Есть ли деньги.** Недавний раунд/грант (`form_d`, `grant`,
   `funding_announcement`), оценка/valuation, признаки подготовки к раунду
   (найм CFO/всплеск вакансий, цитаты «we're raising»). Только `verified`-факты.
2. **Подходящий ли момент.** Свежесть финансового сигнала (≤18 мес — «деньги
   сейчас»), стадия, динамика. Старый сигнал — не повод заходить как «сейчас».
3. **Есть ли потребность в данных.** Модальность, тип разметки, признаки сбора/
   нехватки датасетов из секции product.

→ **Вывод для нас:** насколько вероятна продажа data services + конкретный угол
оффера (что именно предложить под их потребность) + **по каким ЛПР заходить** —
сошлись на собранные `contacts` (tier 1 — бюджет, tier 2 — инициатор потребности).

Для `data_partner` (есть durable-запись `data_partner_flag`): вместо прямой
продажи — **партнёрский угол** (обмен/совместные датасеты, им самим нужны большие
объёмы данных).

Отмечай уровень уверенности и пробелы в данных. Опирайся только на `verified`
факты; `unverified`/`stale` явно называй как требующие проверки.

## Шаг 3 — Записать ноту аудита
```bash
echo '{
  "company_id":"<company_uuid>",
  "section":"audit",
  "facts":{
    "claims_vs_reality":"<...>",
    "market_position":"<...>",
    "forecast":"<...>",
    "money_moment_need":"<деньги + момент + потребность — нарративный синтез>",
    "sales_opportunity":"<вероятность продажи + угол оффера>",
    "contact_angle":"<по каким ЛПР заходить, ссылка на contacts/tier>",
    "needs_verification":["<unverified/stale факты, не вошедшие как факты>"]
  },
  
  "confidence":"<high|medium|low>",
  "model":"claude",
  "version":"v1"
}' | python scripts/dossier_store.py --upsert-analysis-record
```
